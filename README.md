# 1 Description

PepCL is a two-stage deep learning framework for antimicrobial peptide activity prediction by using contrastive learning to refine residue-level peptide representations derived from a pretrained protein language model.

For a given peptide, PepCL first extracts residue-level pretrained features generated by ProtTrans T5-UniRef50. These features are then used to train a contrastive encoder with triplet loss and dynamic hard negative mining. The pretrained contrastive encoder maps the original residue-level features into contrastively refined residue-level representations.

After contrastive pretraining, the downstream prediction model loads the pretrained contrastive encoder checkpoint to generate refined residue-level peptide representations. These representations are further processed by an embedding adaptor, a rotary residual Transformer encoder and an average pooling operation. Finally, the peptide-level representation is inputted into the classification layer for binary activity prediction.

The overall workflow of PepCL contains three main steps:

```text
Step 1: Extract ProtTrans residue-level features
Step 2: Run contrastive encoder pretraining using the code in contrastive_petrain/
Step 3: Train and evaluate the downstream prediction model using the pretrained contrastive encoder checkpoint
```
# 2 Installation

## 2.1 System requirements

For downstream prediction and model training, a GPU is recommended for faster training. CPU-only running is possible for small-scale testing, but training deep models from scratch may be slow.
The environment provided in this repository was built with:

```text
Python 3.8.20
PyTorch 2.4.0
CUDA 11.8
## 2.2 Create an environment

PepCL is built on Python and PyTorch. We recommend using conda to create the running environment from the provided `environment.yml` file.

```bash
conda env create -f environment.yml
conda activate PepCL

Note: Before creating the environment, please check whether the prefix: line exists at the end of environment.yml. If it exists, we recommend deleting this line because it records a local path from the original machine.
## 2.3 Install dependencies manually
The conda environment is recommended. Manual installation is only needed when the provided environment.yml is not suitable for your local CUDA or system configuration.

Please install PyTorch according to your CUDA version. For example:

```bash
# CUDA 11.8
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118

# CPU only
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu

Then install the main required packages:
```bash
pip install numpy h5py scikit-learn prettytable torchnet transformers sentencepiece tqdm

# 4 Prepare pretrained ProtTrans features

PepCL uses residue-level pretrained protein language model features as model input. The current implementation reads pretrained features from H5 files.

## 4.1 Extract ProtTrans T5-UniRef50 features

To generate ProtTrans T5-UniRef50 features, users may refer to ProtTrans:

```text
https://github.com/agemagician/ProtTrans
```

Example command:

```bash
cd {ProtTrans_path}/Embedding

python prott5_embedder.py \
  --input {path_to_fasta_file} \
  --output {path_to_output_h5_file} \
  --model {path_to_ProtT5_model}
```

## 4.2 H5 file naming rule

For both contrastive pretraining and downstream prediction, the H5 feature file should have the same base name as the corresponding TXT file.

Example:

```text
smalltrain.txt  ->  smalltrain.h5
smallvalid.txt  ->  smallvalid.h5
a.txt           ->  a.h5

trainAMP.txt    ->  trainAMP.h5
validAMP.txt    ->  validAMP.h5
testAMP.txt     ->  testAMP.h5
```


# 5 Contrastive encoder pretraining

Before running the downstream prediction model, users should first train the contrastive encoder.

The contrastive pretraining code is provided in this GitHub repository:

```text
contrastive_petrain/
├── dataloader.py
├── main.py
└── model.py
```

The contrastive pretraining module takes three data files as input:

```text
--train_txt    training set used as anchors
--valid_txt    validation set used as anchors
--pool_txt     auxiliary pool used to sample positive and negative peptides
```

Each TXT file should follow a FASTA-like format. Each peptide name should be followed by a binary label separated by a vertical bar `|`.

Example:

```text
>peptide0|1
GLLDTFKNLALNAAKSAGVSVLNSLSKTC
>peptide1|0
METATLVAIFISCLLVSFTGYAPYTASGQPSNELRDLFEEHED
```

The label definition is:

```text
1: positive sample
0: negative sample
```

For each TXT file, the corresponding H5 file should be placed in the same directory and should have the same base name.

Example:

```text
smalltrain.txt  ->  smalltrain.h5
smallvalid.txt  ->  smallvalid.h5
a.txt           ->  a.h5
```
## 5.1 Example: contrastive pretraining for AMP/ABP/Anti-mammalian cells AMP

Assuming that the Zenodo dataset has been downloaded and extracted to the project root directory, the AMP contrastive encoder can be pretrained as follows:

```bash
cd contrastive_petrain

python main.py \
  --train_txt dataAMP/smalltrain.txt \
  --valid_txt dataAMP/smallvalid.txt \
  --pool_txt dataAMP/a.txt \
  --model_name AMP_triplet
 For other two datasets, you can just replace the input fasta

## 5.2 Output of contrastive pretraining

The contrastive pretraining results are saved in:

```text
contrastive_petrain/result/
```

The output files include:

```text
{model_name}_best_{timestamp}.pth
{model_name}_final_{timestamp}.pth
{model_name}_loss_curve_{timestamp}.png
{model_name}_loss_data_{timestamp}.pkl
```

For downstream training, please use the best contrastive encoder checkpoint, for example:

```text
contrastive_petrain/result/AMP_triplet_best_YYYYMMDD_HHMMSS.pth
```

# 6 Downstream prediction model training

After obtaining the pretrained contrastive encoder checkpoint from `contrastive_petrain/result/`, users can train the downstream peptide activity prediction model.

## 6.1 Prepare downstream data files

A recommended directory structure is:

```text
../datasets/AMP/
├── trainAMP.txt
├── validAMP.txt
├── testAMP.txt
└── feature/
    ├── trainAMP.h5
    ├── validAMP.h5
    └── testAMP.h5
```
For ABP and anti-mammalian cell AMP tasks, the file names can be organized in the same way:

```text
../datasets/AMP/
├── trainABP.txt
├── validABP.txt
├── testABP.txt
├── trainMM.txt
├── validMM.txt
├── testMM.txt
└── feature/
    ├── trainABP.h5
    ├── validABP.h5
    ├── testABP.h5
    ├── trainMM.h5
    ├── validMM.h5
    └── testMM.h5
```

## 6.2 Example: downstream training for AMP

```bash
python train.py \
  --type AMP \
  --train_fasta trainAMP.txt \
  --valid_fasta validAMP.txt \
  --test_fasta testAMP.txt \
  --duibi_ckpt contrastive_petrain/result/AMP_triplet_best_YYYYMMDD_HHMMSS.pth \
  --hidden 256 \
  --drop 0.5 \
  --n_transformer 2 \
  --lr 0.0001 \
  --batch_size 128 \
  --epoch 100
```
